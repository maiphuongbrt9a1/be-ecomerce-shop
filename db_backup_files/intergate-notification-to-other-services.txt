**Plan triển khai đề xuất**

1. Chuẩn hóa rule map event -> notification type  
Order/payment/request/return => personal. Voucher campaign => shop.

2. Thêm NotificationService vào các module nghiệp vụ  
OrdersModule, PaymentsModule, RequestsModule, ReturnRequestsModule, VouchersModule.

3. Mỗi service tạo helper nội bộ để bắn notification  
Ví dụ notifyOrderStatusChanged, notifyPaymentConfirmed, notifyReturnRequestUpdated để tránh lặp title/content.

4. Chèn call notification ngay sau khi transaction thành công  
Không bắn trước commit để tránh gửi sai khi rollback.

5. Với cron jobs, thêm cơ chế chống gửi trùng  
Dựa theo orderId + trạng thái mới + mốc thời gian hoặc bảng log notification event.

6. Chuẩn hóa message template  
Tiêu đề ngắn + nội dung có orderId/requestId, trạng thái cũ -> mới, actor (system/staff).

7. Viết test cho các trigger chính  
Ít nhất cho 8 điểm high-priority đầu (orders + payments + return-requests).

Nếu bạn muốn, mình có thể làm luôn bước tiếp theo: tạo ma trận template cụ thể cho từng hàm (title/content tiếng Việt) và patch code theo đúng thứ tự ưu tiên ở trên.
